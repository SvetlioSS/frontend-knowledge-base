## URL encoding
URLs can only be sent over the Internet using the ASCII character-set. If a URL contains characters outside the ASCII set, the URL has to be converted.

URL encoding converts non-ASCII characters into a format that can be transmitted over the Internet.

URL encoding replaces non-ASCII characters with a "%" followed by hexadecimal digits.

URLs cannot contain spaces. URL encoding normally replaces a space with a plus (+) sign, or %20.

### CSS Cascading Order
What style will be used when there is more than one style specified for an HTML element?

All the styles in a page will "cascade" into a new "virtual" style sheet by the following rules, where number one has the highest priority:

Inline style (inside an HTML element)
External and internal style sheets (in the head section)
Browser default
So, an inline style has the highest priority, and will override external and internal styles and browser defaults.

```css

  /* styles for element with class="a" */
  .b {
    /* styles for element with class="b" which is a descendant of class="a" */
  }
  &.b {
    /* styles for element with class="a b" */
  }
}

/* the browser parses this as */
.a {
  /* styles for element with class="a" */
}
.a .b {
  /* styles for element with class="b" which is a descendant of class="a" */
}
.a.b {
  /* styles for element with class="a b" */
}
```

> [!NOTE]
>
> margin auto - pushing element to one or multiple sides

By default, the width and height of an element is calculated like this:

width + padding + border = actual width of an element

height + padding + border = actual height of an element

This means: When you set the width/height of an element, the element often appears bigger than you have set (because the element's border and padding are added to the element's specified width/height)

The `box-sizing` property allows us to include the padding and border in an element's total width and height.

If you set `box-sizing: border-box;` on an element, padding and border are included in the width and height.

This is the current best practice to ensure layouts are predictable in responsive scenarios.

### Different Font Formats

1. <strong>TrueType Fonts (TTF)</strong> - TrueType is a font standard developed in the late 1980s, by Apple and Microsoft. TrueType is the most common font format for both the Mac OS and Microsoft Windows operating systems.
2. <strong>OpenType Fonts (OTF)</strong> - OpenType is a format for scalable computer fonts. It was built on TrueType, and is a registered trademark of Microsoft. OpenType fonts are used commonly today on the major computer platforms.
3. <strong>The Web Open Font Format (WOFF)</strong> - WOFF is a font format for use in web pages. It was developed in 2009, and is now a W3C Recommendation. WOFF is essentially OpenType or TrueType with compression and additional metadata. The goal is to support font distribution from a server to a client over a network with bandwidth constraints.
4. <strong>The Web Open Font Format (WOFF 2.0)</strong> - TrueType/OpenType font that provides better compression than WOFF 1.0.
5. <strong>SVG Fonts/Shapes</strong> - SVG fonts allow SVG to be used as glyphs when displaying text. The SVG 1.1 specification define a font module that allows the creation of fonts within an SVG document. You can also apply CSS to SVG documents, and the @font-face rule can be applied to text in SVG documents.
6. <strong>Embedded OpenType Fonts (EOT)</strong> - EOT fonts are a compact form of OpenType fonts designed by Microsoft for use as embedded fonts on web pages.

### The CSS @font-face Rule
Web fonts allow Web designers to use fonts that are not installed on the user's computer.

When you have found/bought the font you wish to use, just include the font file on your web server, and it will be automatically downloaded to the user when needed.

Your "own" fonts are defined within the CSS @font-face rule.

> [!NOTE]
>
> Use @font-face when:
> * You want a specific font not guaranteed to be on all systems. Example: You're using a rare font like Proxima Nova or a custom-designed one.
> * You need better control over font loading behavior.
> * You can host fonts locally or from a CDN, and manage formats (e.g., WOFF2, TTF) for performance and compatibility.
> * You want to self-host fonts instead of using services like Google Fonts.

### CSS Stacking context
A stacking context is like a self-contained layer of elements on a web page. Elements inside it stack (using z-index) only with each other, not with elements in other stacking contexts. Think of it like each context being its own mini whiteboard on top of the main canvas.

The `z-index` property specifies the stack order of an element within it's stacking context. It's not working across all stacking contexts.

> [!NOTE]
>
> z-index only works on positioned elements (position: absolute, position: relative, position: fixed, or position: sticky) and flex items (elements that are direct children of display: flex elements)

## CSS Object fit
`object-fit` - property is used to specify how an `<img>` or `<video>` should be resized to fit its container.
Possible values are:
* `fill` - This is default. The image is resized to fill the given dimension. If necessary, the image will be stretched or squished to fit
* `contain` - The image keeps its aspect ratio, but is resized to fit within the given dimension
* `cover` - The image keeps its aspect ratio and fills the given dimension. The image will be clipped to fit
* `none` - The image is not resized
* `scale-down` - the image is scaled down to the smallest version of none or contain

`object-position` - property is used to specify how an `<img>` or `<video>` should be positioned within its container.

Example:
```css
img {
  width: 200px;
  height: 300px;
  object-fit: cover;
  object-position: 15% 100%;
}
```

## CSS Variables & Custom properties
Custom properties (sometimes referred to as CSS variables) are entities defined by CSS authors that represent specific values to be reused throughout a document. They are set using the `@property` at-rule or by custom property syntax (e.g., `--primary-color: blue;`). Custom properties are accessed using the CSS var() function (e.g., `color: var(--primary-color);`).

They can have global or local scope.

Global scope example:
```css
:root {
  --blue: #1e90ff;
  --white: #ffffff;
}

body { background-color: var(--blue); }
```

Local scope is on selector level, example:
```css
#div1 {
  --local-blue: #1e90ff;

  background-color: var(--local-blue);
}
```

Variables can be overriden example:
```css
:root {
  --blue: #1e90ff;
  --white: #ffffff;
}

button {
  --blue: #0000ff; /* local variable will override global */
  background-color: var(--white);
  color: var(--blue);
  border: 1px solid var(--blue);
  padding: 5px;
}
```

Variables can be changed with JavaScript like:
```html
<script>
  // Get the root element
  var r = document.querySelector(':root');

  // Create a function for getting a variable value
  function myFunction_get() {
    // Get the styles (properties and values) for the root
    var rs = getComputedStyle(r);
    // Alert the value of the --blue variable
    alert("The value of --blue is: " + rs.getPropertyValue('--blue'));
  }

  // Create a function for setting a variable value
  function myFunction_set() {
    // Set the value of variable --blue to another value (in this case "lightblue")
    r.style.setProperty('--blue', 'lightblue');
  }
</script>
```


## CSS @property Rule
The `@property` rule has data type checking and constraining, sets default values, and defines whether the property can inherit values or not.
```css
@property --background-color {
  syntax: "<color>";
  inherits: false;
  initial-value: blue;
}
```

## CSS Houdini

CSS Houdini is a set of APIs that expose parts of the CSS engine. This makes it easier for developers to create extensions for CSS. These extensions might be to polyfill features that are not yet available in a browser, experiment with new ways of doing layout, or add creative borders or other effects.
